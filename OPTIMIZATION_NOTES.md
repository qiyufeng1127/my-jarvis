## 优化内容总览

本次优化主要针对以下三个方面进行了改进：

### 1. Kiki 问候频率优化 ✅

**优化目标：** 减少不必要的重复打招呼，仅在重要通知时发送消息

**实现方式：**
- 在 `ai-secretary.js` 中添加了智能问候判断逻辑
- 新增 `shouldShowDailyGreeting()` 方法，只在以下情况显示问候：
  - 新的一天开始
  - 当天有待完成任务
  - 周一（每周开始）
- 使用 localStorage 记录上次问候时间，避免频繁打扰

**用户体验改进：**
- 不再频繁显示"下午好"等常规问候
- 只在有实际任务需要提醒时才发送消息
- 减少干扰，提高专注度

### 3. 拖延监控功能增强 ✅

**优化目标：** 将单次铃声改为可循环播放的自定义语音警报

#### 3.1 自定义提示语功能

**新增设置项：**
```javascript
customPreAlertText: "距离{task}开始还有{seconds}秒，请准备启动步骤：{step}"
customAlertText: "任务{task}已超时，请立即开始执行步骤：{step}"
```

**支持的变量：**
- `{task}` - 任务名称
- `{step}` - 步骤名称
- `{seconds}` - 剩余秒数
- `{minutes}` - 已停留分钟数

#### 3.2 循环语音播报

**新增功能：**
- 使用浏览器原生 `speechSynthesis` API 进行语音播报
- 支持循环播放，默认间隔10秒
- 在预警阶段和超时阶段都会播放自定义语音

**配置选项：**
```javascript
useVoiceAlert: true,        // 是否使用语音播报
voiceLoopEnabled: true,     // 是否循环播放
voiceLoopInterval: 10,      // 循环间隔（秒）
```

#### 3.3 金币暂停功能

**新增功能：**
- 支付金币可暂停语音提醒30分钟
- 暂停期间不会播放语音警报
- 30分钟后自动恢复语音提醒

**配置选项：**
```javascript
pauseCost: 10,              // 暂停成本（金币）
pauseDuration: 1800,        // 暂停时长（秒，30分钟）
pauseEnabled: true          // 是否启用暂停功能
```

**使用方法：**
- 在拖延监控面板中点击"💰 暂停提醒"按钮
- 支付10金币即可暂停30分钟
- 系统会显示暂停结束时间
- 超时后若未完成任务，语音警报会自动重新启动

**核心方法：**
- `initSpeechSynthesis()` - 初始化语音合成
- `speakText(text)` - 播放语音文本
- `startVoiceLoop(text)` - 开始循环播放
- `stopVoiceLoop()` - 停止循环播放
- `pauseWithCoins()` - 金币暂停
- `resumeFromPause()` - 从暂停恢复

## 使用建议

### 拖延监控设置建议

1. **启动宽限期：** 建议设置为2-5分钟，给自己足够的准备时间
2. **预警提前时间：** 建议设置为20-30秒，提前做好心理准备
3. **自定义提示语：** 使用激励性语言，如"加油！距离{task}开始还有{seconds}秒"
4. **语音循环间隔：** 建议10-15秒，避免过于频繁

### 低效率监控设置建议

1. **判定时长：** 根据任务复杂度设置，简单任务30分钟，复杂任务60分钟
2. **半程提醒：** 建议开启，及时检查进度
3. **自定义提示语：** 使用提醒性语言，如"已停留{minutes}分钟，需要帮助吗？"
4. **语音循环间隔：** 建议15-20秒，给自己思考时间

### 移动端导航建议

**推荐配置（按使用频率）：**
1. 💬 对话 - 最常用
2. 📅 任务 - 查看任务
3. ⏰ 拖延监控 - 监控进度
4. 🎮 游戏 - 查看奖励
5. ⋯ 更多 - 访问其他功能

**根据工作类型调整：**

## 注意事项

1. **语音播报权限**
   - 首次使用需要用户交互（点击按钮）才能激活
   - 某些浏览器可能需要手动授权

2. **金币消耗**
   - 暂停功能会消耗金币，请合理使用
   - 建议在真正需要专注时使用暂停功能

3. **移动端配置**
   - 自定义导航配置保存在本地
   - 清除浏览器数据会重置配置

4. **性能影响**
   - 语音循环播放对性能影响很小
   - 建议在低电量时关闭语音功能
