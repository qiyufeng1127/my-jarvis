# 修复说明 - 导入导出功能无限循环问题

## 问题原因

启动应用时出现 "Maximum update depth exceeded" 错误，原因是：

1. **forEach 循环触发重渲染**：在导入任务时使用 `forEach` 循环调用 `onTaskCreate`，每次创建任务都会触发组件重新渲染
2. **ID 重复生成**：导入的任务会被重新生成 ID，导致重复任务无法被正确识别

## 修复内容

### 1. 修改 `TimelineCalendar.tsx` 和 `QuickStartView.tsx`
- 将 `forEach` 改为 `Promise.all` + `map`
- 避免循环中的同步状态更新

```typescript
// 修复前
importedTasks.forEach(task => {
  onTaskCreate(task);
});

// 修复后
Promise.all(
  importedTasks.map(task => onTaskCreate(task))
).catch(err => {
  console.error('❌ 批量导入任务失败:', err);
});
```

### 2. 修改 `taskStore.ts`
- 支持导入时保留原有任务 ID
- 只在新建任务时生成 ID

```typescript
// 修复前
id: crypto.randomUUID(),

// 修复后
id: taskData.id || crypto.randomUUID(), // 如果已有ID则使用
```

### 3. 修改 `ImportExportButton.tsx`
- 简化导入数据结构，只传递必要字段
- 让 taskStore 负责填充默认值
- 避免传递可能导致冲突的字段（如 createdAt, updatedAt）

## 测试建议

1. 启动应用，确认不再报错
2. 导出今日任务
3. 导入刚才导出的文件
4. 确认任务正确导入且无重复
