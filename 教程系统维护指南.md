# 📚 教程系统维护指南

## 概述

ManifestOS 的教程系统旨在帮助新用户快速上手，突出 AI 智能化特点。本指南说明如何在功能更新时同步更新教程。

---

## 🎯 教程设计原则

### 1. 突出 AI 智能化
- **强调自动化**：90% 的操作由 AI 完成
- **减少手动操作**：用户只需输入想法
- **展示智能能力**：分类、分解、标签、关联

### 2. 简洁易懂
- **字数精简**：每个功能说明控制在 50-100 字
- **使用示例**：提供具体的使用场景
- **分步骤说明**：1-2-3 清晰步骤

### 3. 渐进式引导
- **首次访问**：欢迎弹窗 + 核心功能引导
- **按需查看**：帮助按钮随时可用
- **不打扰**：可跳过，可重置

## 🔄 更新流程

### 当添加新功能时

#### 步骤 1：更新使用指南 (`UserGuide.tsx`)

**位置**：根据功能类型选择标签页
- **快速开始**：核心流程相关
- **AI 智能核心**：AI 能力相关
- **功能详解**：具体模块功能

**示例**：添加新的 AI 能力

```tsx
// 在 AIContent() 函数中添加新卡片
<div className="bg-white rounded-xl p-5 border-2 border-purple-200">
  <div className="flex items-center space-x-3 mb-3">
    <div className="w-10 h-10 bg-purple-100 rounded-lg flex items-center justify-center">
      <NewIcon className="w-5 h-5 text-purple-600" />
    </div>
    <h4 className="font-bold text-gray-900">新功能名称</h4>
  </div>
  <p className="text-sm text-gray-600 mb-3">
    功能描述（50-100字）
  </p>
  <div className="bg-purple-50 rounded-lg p-3 text-sm">
    <div className="font-semibold text-purple-900 mb-1">示例：</div>
    <div className="text-gray-700">
      输入："用户输入示例"
      <br />
      <span className="text-purple-600">→ AI 处理结果</span>
    </div>
  </div>
</div>
```

#### 步骤 2：更新新手引导 (`OnboardingTooltip.tsx`)

**如果新功能需要引导**，在 `ONBOARDING_STEPS` 中添加：

```tsx
export const ONBOARDING_STEPS = {
  // 现有引导...
  
  // 新功能引导
  newFeature: [
    {
      id: 'step1',
      target: '[data-tour="new-feature-button"]', // CSS 选择器
      title: '🎯 新功能标题',
      description: '这是新功能的说明，告诉用户如何使用...',
      position: 'bottom' as const,
      highlight: true,
    },
    // 更多步骤...
  ],
};
```

**重要**：在对应的组件中添加 `data-tour` 属性：

```tsx
<button data-tour="new-feature-button">
  新功能按钮
</button>
```

#### 步骤 3：更新欢迎弹窗 (`WelcomeModal.tsx`)

**仅当新功能是核心功能时**更新欢迎弹窗：

```tsx
// 在核心亮点部分添加
<li className="flex items-start space-x-2">
  <span className="text-purple-600 font-bold">✓</span>
  <span>新功能的核心价值描述</span>
</li>
```

#### 步骤 4：添加帮助按钮

在新功能的 UI 中添加帮助按钮：

```tsx
import HelpButton from '@/components/tutorial/HelpButton';

// 在组件中使用
<HelpButton
  title="新功能说明"
  content={[
    '第一步：做什么',
    '第二步：做什么',
    '第三步：做什么',
  ]}
  position="bottom"
/>
```

#### 步骤 5：更新文档

在 `更新日志.md` 中记录：

```markdown
## 版本 X.X.X - YYYY-MM-DD

### 🎉 新增功能
- **新功能名称**：功能描述

### 📚 教程更新
- 在使用指南中添加了新功能说明
- 添加了新功能的新手引导
- 更新了帮助提示
```

## 📝 编写规范

### 功能描述

**格式**：
```
[功能名称]：[一句话描述]

使用步骤：
1. 第一步
2. 第二步
3. 第三步

示例：
输入："用户输入"
→ 结果："AI 处理结果"
```

**要求**：
- 描述控制在 50-100 字
- 步骤不超过 4 步
- 必须有具体示例
- 突出 AI 自动化

### 引导文案

**格式**：
```
标题：🎯 [功能名称]
描述：[如何使用]，[AI 会做什么]
```

**要求**：
- 标题加 emoji
- 描述控制在 30-50 字
- 强调 AI 的作用
- 使用友好的语气

### 示例文案

**好的示例**：
```
✅ "对 AI 说出你的想法，它会自动分类并分配到对应模块"
✅ "输入任务列表，AI 会自动分解、排序、优化动线"
✅ "完成任务自动获得金币和成长值奖励"
```

**不好的示例**：
```
❌ "点击按钮，然后选择选项，再点击确认"（太繁琐）
❌ "使用本功能可以提高效率"（太空泛）
❌ "请按照以下步骤操作..."（太正式）
```

## 🎨 设计规范

### 颜色使用

- **紫色**：AI 智能相关
- **蓝色**：任务和时间轴
- **黄色**：金币和奖励
- **绿色**：成功和完成
- **橙色**：目标和进度

### 图标使用

- 🤖 AI 智能
- 📥 收集箱
- 📅 时间轴
- 🎯 目标
- 💰 金币
- 📝 记录
- ✨ 智能功能
- 💡 提示

### 布局规范

- **卡片间距**：4-6 像素
- **内边距**：12-20 像素
- **圆角**：8-16 像素
- **阴影**：subtle，不要太重

## 🔧 常见问题

### Q: 如何决定是否需要新手引导？

**A**: 满足以下条件之一就需要：
- 核心功能（如 AI 输入、收集箱）
- 操作不直观（需要多步骤）
- 新用户容易忽略（如隐藏功能）

### Q: 引导步骤应该有多少个？

**A**: 建议 3-5 个步骤
- 太少：无法完整说明
- 太多：用户会失去耐心

### Q: 如何平衡详细说明和简洁？

**A**: 
- 使用指南：详细完整
- 新手引导：简洁重点
- 帮助按钮：中等详细

### Q: 什么时候更新欢迎弹窗？

**A**: 只在以下情况更新：
- 添加核心功能
- 改变核心流程
- 重大版本更新
