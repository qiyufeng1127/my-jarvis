## 🧠 核心升级：AI指挥中枢

### 新增文件
- `src/services/aiCommandCenter.ts` - AI智能指挥中枢

### 核心能力

#### 1. 深度语义理解
不再依赖关键词，而是真正理解用户的意图

**示例：**
```
用户输入："待会儿去打扫卫生"

旧版（关键词匹配）：
- 检测到"去"、"打扫" → 创建任务
- 机械回复："已创建任务"

新版（语义理解）：
- AI理解："待会儿"表示15-30分钟后
- AI理解："打扫卫生"是家务任务
- AI推断：应该在厕所/客厅等位置
- 智能回复："好的，15分钟后提醒你去打扫卫生。别又找借口拖延啊。"
```

#### 2. 智能意图识别
AI自己判断用户想做什么，不需要固定格式

**支持的意图类型：**
- `create_task` - 创建任务/日程
- `record_mutter` - 记录碎碎念/心情/日记
- `delete_task` - 删除任务
- `move_task` - 移动/修改任务
- `query_info` - 查询信息
- `set_goal` - 设置目标
- `record_income` - 记录收入/支出
- `analyze_mood` - 分析心情
- `remind_me` - 设置提醒
- `chat` - 纯聊天

#### 3. 自动功能调用
AI自己决定需要调用哪些功能，不需要用户指定

**示例：**
```
用户："我今天大姨妈来了好痛"

AI分析：
1. 这不是任务，是碎碎念
2. 表达身体不适
3. 需要记录到记忆库
4. 需要在时间轴创建记录卡片
5. 需要给予关心和建议

AI执行：
- 调用 record_mutter
- 创建时间轴卡片
- 回复："大姨妈来了确实很难受。多喝热水，好好休息，别硬撑。需要我提醒你吃止痛药吗？"
```

#### 4. 上下文理解
AI记住之前的对话，能理解省略和指代

**示例：**
```
用户："明天下午2点开会"
AI："好的，已添加明天下午2点的会议。"

用户："改成3点吧"
AI理解：
- "改"指的是刚才创建的会议
- "3点"是新的时间
- 自动找到那个任务并修改

AI："好的，已经把会议改到3点了。"
```

#### 5. 自然对话
像真人一样聊天，不是机械回复

**示例：**
```
用户："你觉得我今天表现怎么样"

旧版：
"抱歉，我不理解你的意思。"

新版：
AI分析：
- 这是在寻求评价
- 不需要执行功能
- 需要基于今天的数据给出反馈

AI回复：
"今天你完成了3个任务，虽然有点拖延，但最后还是搞定了。不错，继续保持。明天能不能更快点？"
```

## 🔧 技术实现

### 核心流程

```
用户输入
    ↓
AI指挥中枢
    ↓
语义理解（DeepSeek）
    ↓
意图识别
    ↓
决策（需要调用什么功能）
    ↓
执行操作
    ↓
生成回复
    ↓
显示给用户
```

### 关键代码

#### 1. AI指挥中枢调用
```typescript
// 在 handleSend 函数开始处
const intent = await aiCommandCenter.processUserInput(message, context);

// AI自动决定需要执行什么操作
if (intent.actions.length > 0) {
  await aiCommandCenter.executeActions(intent.actions);
}

// 显示AI的智能回复
setMessages(prev => [...prev, {
  role: 'assistant',
  content: intent.reply,
}]);
```

#### 2. 系统提示词
```typescript
const systemPrompt = `
你是${personality.name}，用户的智能AI管家，就像贾维斯一样。

核心能力：

3. 自然对话
4. 上下文理解
5. 情感识别

你能管理的功能：
- 时间轴管理
- 记忆管理

- 智能提醒

重要原则：
1. 理解真实意图，不要被表面文字迷惑
```
